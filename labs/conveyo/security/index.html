Security

Built for a regulated practice: tenant isolation, an audit trail on every action, and no secrets in plain text.

Tenant isolation, three layers deep

Every tenant-scoped record carries an organization id, enforced at the application layer by a tenant-aware data-access layer, again at the database layer via row-level security, and again by role-based permission checks before a request ever reaches the data layer.

Encryption

Connector credentials and other secrets are encrypted at rest and never returned in plain text over the API — read paths redact them by default.

Audit trail

An append-only audit log records who did what, when, on every matter, document, trust transaction, and connector run — the evidentiary record a regulated conveyancing practice needs.

Canadian data residency options

Where a connected provider supports it — including document-processing regions — data can be kept in Canadian regions rather than defaulting to a US-only footprint.
