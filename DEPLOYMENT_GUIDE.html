Step 1: Create Google Sheet Backend ~5 min

1
Create a new Google Sheet

Go to sheets.google.com (logged in as homeswithmanish@gmail.com). Create a blank spreadsheet and name it "Website Leads".

2
Copy the Sheet ID

The Sheet ID is in the URL: docs.google.com/spreadsheets/d/THIS_PART/edit. Copy it — you'll need it in Step 2.

3
Open Apps Script

In your Google Sheet, go to Extensions → Apps Script. This opens the script editor attached to your sheet.

4
Paste the backend code

Delete the default function myFunction() code. Open Google_Sheets_Backend.js from the website folder, copy all contents, and paste it into the Apps Script editor.

5
Set your Sheet ID

Near the top of the code, find const SHEET_ID = 'YOUR_SHEET_ID_HERE'; and replace it with your actual Sheet ID from step 2.

6
Run initial setup

Click the function dropdown (next to the play button) → select createInitialSheet → click Run. Authorize when prompted. This creates the "Leads" sheet with proper headers.

Then run setAdminKey. Check the Execution Log (View → Logs) — it will show your admin API key. Copy and save this key securely.

Important: The admin API key is only shown once in the logs. Save it somewhere safe — you'll need it for the admin dashboard.

Step 2: Deploy Apps Script as Web App ~3 min

1
Click Deploy → New deployment

In the Apps Script editor, click the Deploy button (top right) → New deployment.

2
Configure the deployment

Click the gear icon → select Web app.

Set these options:

• Description: "Leads Backend v1"

• Execute as: Me (homeswithmanish@gmail.com)

• Who has access: Anyone

Click Deploy.

3
Copy the Web App URL

You'll see a URL like: https://script.google.com/macros/s/AKfyc.../exec

Copy this URL. This is your backend endpoint.

Note: "Anyone" access means anyone can submit the form, but only requests with the admin API key can read leads. The form has rate limiting built in.

Step 3: Push to GitHub ~5 min

Update the website with your URLs

In index.html: Find: YOUR_GOOGLE_SCRIPT_URL Replace with: your Apps Script web app URL from Step 2 In admin.html: Find: YOUR_GOOGLE_SCRIPT_URL Replace with: same Apps Script URL Find: YOUR_ADMIN_API_KEY Replace with: your admin key from Step 1
2
Create GitHub repo

Go to github.com/new. Create a new repo named homeswithmanish.github.io (or any name). Set it to Public.

3
Push the website folder
cd /path/to/REALTOR_LEAD_SYSTEM/website git init git add . git commit -m "Initial website launch" git branch -M main git remote add origin https://github.com/homeswithmanish/homeswithmanish.github.io.git git push -u origin main
Before pushing: Make sure Google_Sheets_Backend.js doesn't contain your actual Sheet ID or admin key. The file is for reference only — the actual script lives in Google Apps Script. Consider adding it to .gitignore if you've added sensitive values.

Step 4: Enable GitHub Pages ~2 min

1
Go to repo Settings

2
Configure Pages

In the left sidebar, click Pages.

3
Verify deployment

Go to Actions tab in your repo. You should see a workflow running. Once it completes (green check), your site will be live.

Step 5: Connect Your Domain ~10 min

1
Add custom domain in GitHub Pages

In Settings → Pages → Custom domain, type homeswithmanish.com and click Save.

The CNAME file is already in the repo.

2
Update DNS at your domain registrar

Log into wherever your domain homeswithmanish.com is registered. Go to DNS settings and add these records:

Type: A Name: @ (or leave blank) Value: 185.199.108.153 Type: A Name: @ (or leave blank) Value: 185.199.109.153 Type: A Name: @ (or leave blank) Value: 185.199.110.153 Type: A Name: @ (or leave blank) Value: 185.199.111.153 Type: CNAME Name: www Value: homeswithmanish.github.io
3
Enable HTTPS

Back in GitHub Settings → Pages, check "Enforce HTTPS". This may take a few minutes to provision the SSL certificate.

DNS Propagation: Changes can take 15 minutes to 48 hours. Typically works within 30 minutes. You can check progress at dnschecker.org.

Step 6: Verify CI/CD Pipeline ~2 min

How the CI/CD Works

Every time you push code to the main branch, GitHub Actions automatically:

1. Runs 99 validation tests (HTML structure, SEO, accessibility, security, etc.)

2. Checks file sizes for performance

3. Validates HTML tag structure

4. Checks for broken internal links

5. If all tests pass → deploys to GitHub Pages

6. If any test fails → deployment is blocked, you'll see a red X on the Actions tab

1
Make a test change

Edit any file (like adding a word to a testimonial), commit, and push. Watch the Actions tab to see the pipeline run.
